Hardware Architecture Block Diagram

ATM Controller

Main processing board managing all I/O, security logic, and banking network protocols.

Encrypted PIN Pad

Tamper-resistant keypad that hardware-encrypts PINs before they ever reach the CPU.

Card Reader

Detects EMV chips or magnetic stripes. Includes fraud sensors for anti-skimming.

Cash Dispenser

Mechatronic system with rollers and sensors to pick, count, and dispense currency notes.

Secure Vault

UL-rated steel safe containing the cash cassettes and high-security dispensers.

Receipt Printer

Thermal printer for transaction records, connected via internal USB/Serial.

Software Architecture Stack
APPLICATION: Transaction State Manager, UI, Security Logic
MIDDLEWARE: XFS/CEN API - Hardware Abstraction Layer

Transaction Logic
  • Atomicity: Transactions are all-or-nothing to prevent balance mismatches.
  • ISO 8583: Standard messaging protocol for banking host communication.
  • HSM Integration: Hardware security module calls for key rotation and PIN validation.
Diagnostic Error Segments

Trap State (Automata)

Occurs when an invalid sequence (like a wrong PIN) or out-of-order operation is entered. System enters a security lock ($q_{dead}$).

Logic Failure

I/O Mechanical Jam

Sensors in the dispenser detect stuck notes. Triggers immediate halt and error log.

Hardware Fault

HSM Tamper Event

Internal hardware sensors detect physical case breach. Wipes encryption keys instantly.

Security Breach

Network Timeout

Latency in Bank Host response exceeding 30s. Safely reverts transaction states.
