NFA and ε-NFA

NFA

In a Non-Deterministic Finite Automaton, one input may lead to multiple next states. In this ATM model, entering a PIN can move the system toward successful authentication or toward a security dead state.

q1 --pin--> {q2, q_dead}
q2 --op--> {q3, q_dead}
q3 --withdraw--> {q4}

ε-NFA

An epsilon transition changes state without consuming input. Here, once the card is inserted, the system can automatically prepare the next logical configuration using ε-transition behavior.

q1 --ε--> q2
The machine moves forward without consuming another input symbol.

Theory of Computation Used in This Project

1. DFA in the ATM Flow

The original project is a deterministic finite automaton. Each valid input from a current state has exactly one next state, so the machine follows a fixed sequence from card insertion to exit.

2. NFA Extension

We extended the project with non-deterministic behavior to explain how one symbol can represent multiple conceptual outcomes. PIN entry is the key example because it can be interpreted as success or failure.

3. ε-NFA Usage

The ε-transition is used to describe automatic system movement, such as preparing the next state without consuming a visible user input. This is useful for explaining hidden control logic in the ATM.

4. NFA to DFA Conversion

The project demonstrates subset construction by combining multiple NFA states into one DFA state. This removes non-determinism while preserving the accepted language.

5. Regular Expression

The transaction pattern (card)(pin)(op)(withdraw)(exit) summarizes the legal symbol sequence. The project includes a regex checker to validate this workflow.

6. Context-Free Grammar

The ATM sequence is also written as production rules. This gives a derivation-based representation and shows how a valid transaction string is generated step by step.

7. PDA

Pushdown automata add stack memory. In this project, the stack stores session markers such as card and PIN, which helps explain memory beyond finite states.

8. Turing Machine

The Turing Machine model uses a tape of ATM actions and a head that reads one cell at a time. This shows how the workflow can be interpreted using a general model of computation.

DFA: exact next state
NFA: multiple next states
ε-NFA: automatic transitions
PDA: stack memory
TM: tape + head
